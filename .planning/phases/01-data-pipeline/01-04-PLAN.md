<objective>
Build the icon slice: resolve each tech's source `.dds` icon (respecting the `icon=` override, the `tech_<key>.dds` naming convention, and `technology_swap` aliases), convert it to a web-ready lossless WebP (DDS -> PNG via ImageMagick, PNG -> WebP via sharp), and fall back to a shipped placeholder when a tech has no icon. This completes DATA-04's data content; assemble.ts (Plan 05) drives it over every tech and writes to `data/v{version}/icons/`.

Purpose: Every tech must have a web-ready icon referenced from the snapshot (DATA-04). RESEARCH.md already ran the exact D-12 smoke test — ImageMagick 7.1.1 converts all observed compression formats (uncompressed, DXT1, DXT3, DXT5) cleanly, so the texconv fallback is a documented contingency, not an expected step. This plan also owns the phase's one security-sensitive surface: invoking an external CLI with a user-configurable install path.

Output: `resolveIconSource`, `convertDdsToWebp`, and a shipped `placeholder-icon.webp`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-configurable install path -> ImageMagick CLI | gameRoot-derived icon paths (D-15) are passed to an external process — the phase's only command-execution surface |
| .dds asset -> conversion pipeline | Externally-authored binary image files are decoded by ImageMagick |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering (command injection) | ImageMagick invocation with a configurable install path | mitigate | Invoke via `execFileSync('magick', [ddsPath, pngTempPath])` with an ARGUMENT ARRAY and no `shell: true` — paths are never concatenated into a shell string, so path contents (spaces, metacharacters) cannot break out into command execution |
| T-04-02 | Denial of Service | A malformed/huge .dds crashing or hanging ImageMagick | accept | Local, fixed, trusted corpus (864 game-shipped .dds files); build-time single-user tool; a per-icon conversion failure is caught and falls back to the placeholder (D-13) rather than hanging the whole build |
| T-04-03 | Availability | ImageMagick not installed / DDS coder absent on a future machine | mitigate | RESEARCH.md verified magick 7.1.1 with DDS/DXT coders is installed; convert.ts surfaces a clear error if `magick` is not on PATH; texconv documented as the contingency fallback per D-12 |
| T-04-SC | Tampering | npm installs | mitigate | Only sharp is used here, verified [VERIFIED: npm registry] in Plan 01's audit; no new packages |
