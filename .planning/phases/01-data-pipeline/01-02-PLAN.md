- `Tech` type — the target shape the extractor must produce (fields: key, area, category[], tier, cost, costRaw?, weight, weightModifierRaw?, prerequisites[], unlocks, dlc, flags{isRare,isDangerous,isRepeatable,isStarting}, name, description, icon).

Verified corpus facts (RESEARCH.md — all counted directly against real 4.5.0 files):
- 33 tech files in common/technology/*.txt (plus category/ and tier/ subdirs which are NOT tech files — skip subdir contents when enumerating tech files).
- 678 top-level `tech_*` definitions total. `000_documentation.txt` parses to 0 tech keys (a doc stub) — filter it out. Only keys matching `tech_*` are techs; skip non-tech top-level keys.
- Flags in the raw data: `is_rare = yes`, `is_dangerous = yes`, `start_tech = yes` (isStarting), and repeatable techs use `levels = -1` (or a positive levels value in *_repeatable.txt files) — isRepeatable is true when a `levels` field is present.
- `cost`/`weight` are usually bare numbers or `@variable` strings; block-form `cost = { factor... }` is documented-but-unobserved (Open Question 1 / A3) — handle both: if cost/weight is an object, store it in costRaw/weightModifierRaw and derive the numeric field defensively (use factor if present, else 0) rather than throwing.
- `weight_modifier` blocks: preserve raw; duplicate `modifier` children auto-array via jomini (D-06) — do NOT collapse.
- `technology_swap`: 207 blocks across 18 files; singular-or-array (Pitfall 5) — normalizeToArray before use.

Verified DLC facts (RESEARCH.md Pitfall 4 + Don't Hand-Roll):
- `dlc/dlc0XX_*/dlc0XX.dlc` files (30 folders) are Clausewitz format with a `name = "..."` field (e.g. `name = "Ancient Relics Story Pack"`) that is the EXACT string used by in-script `host_has_dlc = "..."` triggers. Parse them with parseClausewitzFile.
- Primary DLC tag: filename convention (`00_ancient_relics_tech.txt` -> Ancient Relics), cross-checked to the .dlc registry names.
- Secondary/override: scan each tech's `potential` block for `host_has_dlc = "..."`. Only 11 occurrences across 3 files; exactly 2 techs are genuinely cross-DLC (`00_eng_tech.txt` -> Apocalypse and Federations, `00_strategic_resources_tech.txt` -> Distant Stars Story Pack). host_has_dlc, when present, OVERRIDES the filename tag.
- Base-game techs (no DLC filename match, no host_has_dlc) get dlc = null.

Verified graph facts:
- `prerequisites = { "tech_x" "tech_y" }` — array of quoted keys (use normalizeToArray). A dangling prerequisite (references a key not in the extracted set) is a structural error -> fail loud (D-16). Reverse edges ("leads to") = for each tech, the set of techs that list it as a prerequisite (D-05 computed unlocks). The graph must be acyclic — a cycle is a structural error.
