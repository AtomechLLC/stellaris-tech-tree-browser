# Quick Task 260708-3zu — Category bands + tints + re-pack + dark header

## Why

Three user asks:
1. **Tinted category bands** behind the tree (computing, field_manip, …). The
   current ELK layout INTERLEAVES categories (verified: every category spans the
   full ~30,910px height), so there are no bands to tint. → Re-band the layout so
   each category is a contiguous horizontal row, then draw a tinted background
   per band.
2. **A "re-pack layout" button** so the tree isn't spread out when categories are
   toggled off (re-run the layout on only the visible categories).
3. **Dark the top bar** to match the dark tree (it's currently a white strip).

## Task 1 — Banded layout (`app/src/lib/tree/layoutTree.ts`)

Change `layoutTree` to produce **category swimlane bands** and accept an optional
active-category filter:

`layoutTree(snapshot, cardW, cardH, activeCategories?: Set<string>): Promise<TreeLayout>`

- Filter techs to `activeCategories` (default = all) BEFORE building the ELK graph
  (so "re-pack" on a subset lays out only those, and bands = only active cats).
- Run ELK exactly as today (keep all current options — layered, RIGHT, tier
  partition, thoroughness 1) to get a good **y-ORDER** (crossing sense) per node.
  Keep the dangling-prereq guard.
- **Remap into bands** (import `CATEGORY_ORDER`, `CATEGORY_AREA` from
  `../graph/categories`). Constants: `COL_W = cardW + 110`, `ROW_H = cardH + 20`,
  `BAND_LABEL_H = 30`, `BAND_GAP = 44`.
  - Group filtered nodes by `tech.category[0]`, iterate in `CATEGORY_ORDER`
    (skip empty categories).
  - `cursorY` starts at 0. For each category band:
    - Band content starts at `cursorY + BAND_LABEL_H` (top strip reserved for the
      band label).
    - For each tier 0..5: take the band's nodes in that tier, **sort by their ELK
      y**, and stack them: `x = tier * COL_W`, `y = contentTop + rowIndex * ROW_H`.
    - `contentH = max over tiers of (nodesInTier.length * ROW_H)`.
    - Push band geometry `{ category, area, label, top: cursorY, height:
      BAND_LABEL_H + contentH + 12 }`.
    - `cursorY = band.top + band.height + BAND_GAP`.
  - `width = 6 * COL_W`; `height = cursorY`.
- **Edges:** positions changed from ELK's, so ELK's bend points are stale — return
  every edge with `sections: []`. (The existing `EdgeLayer` fallback draws a clean
  source-right → target-left elbow between the new positions.)
- Extend `TreeLayout` with `bands: BandGeometry[]` and export
  `interface BandGeometry { category: string; area: string; label: string; top: number; height: number; }`.
  Use `categoryLabel(category)` for `label`.

**Verify:** unit test — nodes grouped so each category occupies a contiguous,
non-overlapping y-range; `bands.length` = number of non-empty categories; x is a
multiple of COL_W (tier-aligned); area order preserved (physics bands above
society above engineering). `npm run build`.

## Task 2 — Tinted band backgrounds (`app/src/components/BandLayer.tsx` NEW)

A memoized layer rendered as the FIRST child of `.tree-canvas` (behind the edge
layer + cards, z-index 0). Props: `{ bands: BandGeometry[]; width: number }`.
For each band render an absolutely-positioned `.band` rect (`top: band.top`,
`height: band.height`, `width`), plus a `.band__label` (the category name) at the
band's top-left. `data-area` on each for area coloring. Alternate a
`data-alt={index % 2}` so adjacent bands are distinguishable.

CSS (`app/src/styles/app.css`):
- `.band-layer { position:absolute; inset:0; z-index:0; pointer-events:none; }`
- `.band { position:absolute; left:0; }`
- Tint by AREA color, faint, alternating: e.g.
  `.band[data-area="physics"] { background: color-mix(in srgb, var(--area-physics) 8%, transparent); }`
  and `.band[data-area="physics"][data-alt="1"] { background: color-mix(in srgb, var(--area-physics) 14%, transparent); }`
  (same pattern for society/engineering). Keep it subtle so cards/text stay
  readable.
- `.band__label { position:absolute; left:10px; top:6px; font: heading; font-size:
  13px; letter-spacing:.02em; opacity:.85; }` colored by area (data-area →
  `--area-*`). Uppercase optional.
- Cards already have `z-index:2`, edges `z-index:1` — band layer at 0 sits behind
  both. Confirm the cards/edges still render above.

**Verify:** dev server — faint area-tinted bands behind each category row, with
category labels; bands align to the card rows; cards + dashed edges above.

## Task 3 — Re-pack button + wiring (`app/src/components/TechTree.tsx`)

- Render `<BandLayer bands={filteredBands} width={layout.width} />` as the first
  child inside `.tree-canvas`. Filter bands to the active set in the `filtered`
  memo (add `bands: layout.bands.filter(b => active.has(b.category))`) so
  toggled-off bands' backgrounds disappear too (leaving a gap until re-pack).
- Add a **"Re-pack" button** (top-right of the viewport, or beside the zoom
  controls). On click: re-run `layoutTree(snapshot, CARD_W, CARD_H, active)` and
  swap `state.layout` when it resolves — WITHOUT unmounting the tree (keep status
  "ready", show a small "Re-packing…" busy state on the button; ignore stale
  results if active changed again). After it resolves, `resetView()` (or fit) so
  the newly-packed tree is framed. Because re-pack lays out only active
  categories, their bands stack with no gaps.
- Keep everything else (imperative transform, LOD, tooltips, nav). The `content`
  memo's card/edge elements already key off the (filtered) layout — fine.
- NOTE: `layoutTree`'s new 4th arg is optional; the initial effect can keep
  calling `layoutTree(snapshot, CARD_W, CARD_H)` (all categories) — the tree is
  banded from first paint.

**Verify:** dev server — toggle a category off → its band/cards vanish leaving a
gap; click Re-pack → remaining bands close up compactly and the view reframes.
`npm run build`.
