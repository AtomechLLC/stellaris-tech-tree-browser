# Quick Task 260707-we6 — Category swimlanes + reference-style tech cards

## Goal

Two user-requested improvements to the Phase 2 tech-tree render (`app/`):

1. **Swimlanes** — subdivide the current 3 area bands into **13 category lanes**
   (industry, materials, propulsion, …) nested within the areas, with left-side
   lane labels + subtle lane backgrounds.
2. **Reference-style tech cards** — replace the small Sigma icon-nodes, when
   zoomed in, with rich HTML cards matching the community reference tool: icon
   panel + tier badge, an **area-colored title header** with the tech name, and
   `Category – Tier N` + `Cost: X, Weight: Y` fields inside a bordered block.
   **Zoom-based LOD:** zoomed out → compact Sigma tiles (fast, whole-tree
   scannable); zoomed in → full HTML cards for on-screen nodes only.

## Locked design decisions (do NOT revisit)

- Swimlanes **nested within** the 3 areas, area order top→bottom: physics,
  society, engineering.
- Node colors stay **by AREA** (physics `#0072b2` blue, society `#e69f00`
  orange, engineering `#d55e00` vermillion) — the card header/border is colored
  by area. Sourced via `theme.ts` token bridge, never hardcoded hex.
- **Zoom-based LOD** for cards (compact tiles out, full cards in).
- Keep Sigma (WebGL) for layout + edges + camera + the zoomed-out overview.
  Cards are an **HTML overlay on top of Sigma**, not a renderer rewrite.

## Data facts (verified against `app/public/data/v4.5.0/tech.json`)

- Every tech has exactly ONE category (`Tech.category: string[]`, all 678 length
  1). Use `category[0]`.
- 13 categories nested under 3 areas (counts): physics = computing(70),
  field_manipulation(42), particles(55); society = biology(121), statecraft(77),
  military_theory(34), new_worlds(32), archaeostudies(24), psionics(24);
  engineering = industry(52), materials(42), propulsion(41), voidcraft(64).
- `Tech` fields available for cards: `name`, `category`, `tier`, `cost`,
  `weight`, `area`, `icon`. (See `pipeline/src/schema/tech-snapshot.ts`.)

## Task 1 — Data + swimlane layout (the layout backbone)

**Files:** `app/src/lib/graph/buildGraph.ts`, `app/src/lib/graph/areaBands.ts`
(rename to `swimlanes.ts`), `app/src/lib/graph/layout.ts`,
`app/src/lib/graph/categories.ts` (NEW shared constant), `app/src/test/layout.test.ts`,
`app/src/test/smoke.test.ts` (if it references bands).

**Action:**
- NEW `categories.ts`: export `CATEGORY_ORDER` (grouped by area, the exact 13-lane
  order): physics → `computing, field_manipulation, particles`; society →
  `biology, statecraft, military_theory, new_worlds, archaeostudies, psionics`;
  engineering → `industry, materials, propulsion, voidcraft`. Also export
  `CATEGORY_LABEL` (key → display name, Title Case, e.g. `field_manipulation` →

  13 lanes ordered by `CATEGORY_ORDER`; within each lane, stable-sort by ELK's

  HEIGHT scales to member count with a MIN floor (e.g. `height = max(MIN_LANE,

  aren't stretched. Small gap between lanes; larger gap between areas. `x`
  untouched (authoritative from ELK). **Export the computed lane geometry**
  (array of `{ category, area, top, height, center }` in graph-space) — the lane
  axis + backgrounds consume it. Provide it either as a return value or a
  module-level getter the layout stores; Task 3 will read it.
- `layout.ts`: update the import + the doc comment ("area-band" → "category
  swimlane") and pass through the exported lane geometry.
- Tests: update `layout.test.ts` to assert swimlane behavior — each node lands in
  its category lane; area order preserved (all physics categories' y < all
  society < all engineering); lanes non-overlapping in y; `x` equals ELK's x.

**Verify:** `cd app && npm test` (green). `npm run build` clean.
**Done:** category flows onto nodes; 13 nested lanes computed; lane geometry

## Task 3 — HTML card overlay with zoom-LOD (the headline)

**Files:** NEW `app/src/components/TechCardOverlay.tsx`, wire into
`app/src/components/TechTreeCanvas.tsx`, CSS in the app stylesheet (where
`.tier-axis` lives — find it), reuse `categories.ts` labels.

**Action:** A React component inside `<SigmaContainer>` (so `useSigma()` works)
that renders an absolutely-positioned HTML layer over the canvas:
- Subscribe to Sigma `camera "updated"` + `resize` (rAF-coalesced, like
  `TierAxis.tsx`). On each update, read `camera.ratio`.
- **LOD gate:** if `ratio >= RATIO_THRESHOLD` (zoomed OUT), render NOTHING (the
  Sigma tiles show). If `ratio < RATIO_THRESHOLD` (zoomed IN), render cards.
  Tune `RATIO_THRESHOLD` against the real render so cards appear at a comfortable
  reading zoom (verify in dev server).
- **Culling:** when active, compute the on-screen node set — for each node,
  `sigma.graphToViewport(nodeAttrs)`; keep only nodes whose viewport position is
  within the container rect + a margin. Only these get card DOM (expect tens, not
  678). Iterate `graph.forEachNode` or maintain the node list from props.
- **Card DOM** (match the reference), per visible node, positioned at its
  viewport x/y (translate; center the card on the node), scaled with zoom
  (e.g. `transform: scale(k)` where k derives from ratio, clamped) so cards feel
  anchored in the graph:
  ```
  <div class="tech-card" data-area={area}>
    <div class="tech-card__icon"><img src={image} alt=""/><span class="tech-card__tier">{roman(tier)}</span></div>
    <div class="tech-card__body">
      <div class="tech-card__title">{name}</div>
      <div class="tech-card__meta">{CATEGORY_LABEL[category]} – Tier {tier}</div>
      <div class="tech-card__stats">Cost: {cost}, Weight: {weight}</div>
    </div>
  </div>
  ```
  Title/`name` is PLAIN TEXT (never innerHTML — D-05 XSS contract). `roman(tier)`
  = I..VI for tier 0..5 (or show the tier number).
- **CSS:** header + border colored by area via `data-area` → the bridged area
  colors (physics blue / society orange / engineering vermillion). Bordered
  rounded block, icon panel left, header bar with the title, muted meta + stats
  lines. Keep it readable; the reference screenshot is the visual target
  (icon+tier badge, colored header, `Category – Tier`, `Cost, Weight`). Pointer
  events: let clicks/pans still reach Sigma (overlay `pointer-events: none` on the
  layer; cards themselves can be `pointer-events: auto` if interactive later — for
  now none needed).
- Wire `<TechCardOverlay graph={graph} />` into `TechTreeCanvas` next to
  `TierAxis`. The overlay must NOT interfere with pan/zoom.

**Perf guard:** only visible nodes get DOM; recompute on camera update, throttled
to one rAF. `log`/note the visible-count if helpful. No per-frame layout of 678
nodes.

**Verify:** dev server — zoom OUT: compact tiles, no cards, smooth pan/zoom; zoom
IN: reference-style cards appear for on-screen techs with correct name/category/

**Done:** zoom-LOD card overlay works and matches the reference presentation.
