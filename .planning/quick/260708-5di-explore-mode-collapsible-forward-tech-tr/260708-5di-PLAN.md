# Quick Task 260708-5di — Explore mode (UC2)

A second view alongside the swimlane Map: a Map ↔ Explore toggle. Explore opens
collapsed at the entry-point techs and you expand one at a time to see what each
unlocks in the next tier (use case 2). Reuses the existing viewport / pan-zoom /
cards / edges / tooltip / selection — only the layout + an expand affordance are
new.

## Task 1 — Explore layout (pure, synchronous) — `app/src/lib/tree/exploreLayout.ts` NEW

`layoutExplore(snapshot, expandedKeys, active, cardW, cardH): TreeLayout` (same
`TreeLayout` shape as `layoutTree`, `bands: []`). NO ELK — a plain
collapsible-tree layout, recomputed on each expand/collapse (cheap).

- Precompute once (module-level memo keyed by snapshot is fine, or inside):
  `childrenByKey` = reverse-prereq map (tech → techs whose `prerequisites`
  include it), and `roots` = techs with `prerequisites.length === 0`, sorted by
  `(tier, categoryIndex, name)`.
- Respect the category filter: a tech is "shown-eligible" only if
  `active.has(category)` (active default = all). Filter roots + children to
  shown-eligible.
- Build the visible forest by DFS pre-order with a GLOBAL `visited` set (dedup —
  the DAG is rendered as a spanning tree; each tech appears once, at its first
  reveal): start from roots; for each node, if it is in `expandedKeys`, recurse
  into its shown-eligible children (skipping already-visited).
- Position: each visited node = one ROW. `x = depth * COL_W`, `y = rowIndex *
  ROW_H` (COL_W = cardW + 90, ROW_H = cardH + 16). Edges = parent→child pairs in
  the visible tree, `sections: []` (EdgeLayer draws elbows).
- Also return, per node, whether it is EXPANDABLE (has ≥1 shown-eligible child)
  and EXPANDED (in expandedKeys) — carry these on the LayoutNode (add optional
  `expandable?: boolean; expanded?: boolean` to `LayoutNode`, unused by the map
  layout) so the renderer can show the chevron. `width`/`height` = extent.

**Verify:** unit test — roots present when collapsed; expanding a root adds its
children as rows to its right; dedup (a multi-parent tech appears once); collapse
removes descendants.

## Task 2 — Expand affordance on the card — `TechCard.tsx`, CSS

Add optional props `expandable?`, `expanded?`, `onToggleExpand?: (key) => void`.
When `expandable`, render a small chevron toggle (▸ collapsed / ▾ expanded) on the
card (e.g. right edge, vertically centered). Its `onClick` calls
`onToggleExpand(tech.key)` and `stopPropagation()` (so expanding does NOT select).
The chevron only renders when `expandable` is set — map mode passes nothing, so
cards are unchanged there.

## Task 3 — View toggle + wire Explore — `TechTree.tsx`, CSS

- Add `viewMode: "map" | "explore"` state and `expandedKeys: Set<string>` state.
- A **Map / Explore** toggle (segmented control) near the Re-pack button (top of
  the viewport). Switching to Explore fits/reset the view to the explore roots;
  switching back restores the map.
- When `viewMode === "explore"`: compute `layoutExplore(snapshot, expandedKeys,
  active, CARD_W, CARD_H)` (useMemo on `[snapshot, expandedKeys, active]`) and use
  it as the layout for the canvas — render the SAME cards + EdgeLayer, but NO
  BandLayer/watermark (bands are map-only). Pass each card `expandable`/`expanded`
  from its LayoutNode + a stable `onToggleExpand`.
- `onToggleExpand(key)`: if expanded, remove `key` AND all its (now-hidden)
  descendants from `expandedKeys`; else add `key`. Selection/hover/highlight/pan/
  zoom all keep working (edges highlight for the selected node the same way).
- The Re-pack button is map-only (hide it in explore). CategoryNav filter still
  applies (filters shown-eligible techs). The ancestry panel is map-only or
  harmless in explore — keep it simple, prefer map-only.

**Verify:** dev server — toggle to Explore: a collapsed column of entry-point
techs; each expandable card shows a chevron; clicking it reveals that tech's
unlocks to the right with connectors; collapsing hides them; hover tooltip +
click-select + gold edge highlight still work; toggling back shows the swimlane
map unchanged.
