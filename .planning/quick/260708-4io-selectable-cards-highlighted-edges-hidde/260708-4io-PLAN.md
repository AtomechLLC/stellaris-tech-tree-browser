# Quick Task 260708-4io — Card selection, edge highlight, ancestry drill-down

Three connected features on the DOM tech tree. Selection state is the spine.

## Task 1 — Selectable ("targetable") cards (`TechTree.tsx`, `TechCard.tsx`, CSS)

- Add `selectedKey: string | null` state in `TechTree`. Add a `movedRef` (a
  ref<boolean>) set true in `onPointerMove` when the drag moves > ~4px from the
  pointerdown origin, reset false in `onPointerDown` — so a drag doesn't count as
  a click.
- `onSelect = useCallback((key) => { if (movedRef.current) return; setSelectedKey(k => k === key ? null : key); }, [])` — click toggles selection, suppressed after a drag.
- Deselect: `Escape` key (window listener in an effect) and clicking empty
  viewport background (a viewport `onClick` that deselects when
  `e.target.closest('.tech-card')` is null) both clear selection.
- `TechCard`: add props `selected?: boolean`, `onSelect?: (key: string) => void`
  (both stable). `onClick={() => onSelect?.(tech.key)}`. Set `data-selected` when
  selected, and `data-key={tech.key}`.
- In the `content` useMemo, pass `selected={node.key === selectedKey}` and
  `onSelect` to each `TechCard`, and ADD `selectedKey` to the memo deps. (Cards
  are `React.memo`, so on a selection change only the 2 changed cards re-render;
  hover/pan don't touch `selectedKey` so they stay cheap.)
- CSS: `.tech-card[data-selected]` gets a clear target highlight — a bright
  outline + glow (e.g. `outline: 2px solid var(--color-select); box-shadow: 0 0
  0 3px color-mix(in srgb, var(--color-select) 45%, transparent);`). Define a
  `--color-select` token (a gold/yellow, e.g. `#ffd23f`) in tokens.css (or the
  :root block) so it's not a bare hex.

**Verify:** dev server — clicking a card highlights it; clicking it again or
Escape or empty space clears it; dragging does NOT select.

## Task 2 — Highlight the selected card's edges (`EdgeLayer.tsx`, CSS)

- `EdgeLayer` gets a new prop `selectedKey?: string | null` (add to its
  React.memo props; pass from TechTree's `content` memo).
- Keep the existing single combined base `<path>` (all edges, dim/dashed) —
  memoized on `[edges, nodes]`.
- ADD a second **highlight** `<path>`: the `d` for edges where `from ===
  selectedKey || to === selectedKey` (i.e. the selected tech's prerequisites AND
  its children/leadsTo), built with the same elbow routing, memoized on
  `[edges, nodes, selectedKey]`. Render it AFTER (on top of) the base path with
  class `edge-layer__path--highlight`.
- CSS: `.edge-layer__path--highlight { stroke: var(--color-select); stroke-width:
  3; stroke-dasharray: none; opacity: 1; }` — solid, thick, yellow (per request).

**Verify:** dev server — selecting a card draws solid thick yellow lines to its
prereqs and children; dashed grey elsewhere. Deselect returns to all-dashed.

## Task 3 — Hidden-ancestry drill-down panel (`AncestryPanel.tsx` NEW, wired in `TechTree.tsx`, CSS)

When a card is selected AND at least one of its recursive prerequisites is HIDDEN
by the current category filter, show a panel with the tech's ENTIRE upstream
inheritance tree so hidden dependencies are still visible/hoverable/selectable.

- Compute (in TechTree or the panel, memoized on `[selectedKey, active,
  techByKey]`): the full recursive ancestry of the selected tech (walk
  `tech.prerequisites` transitively via `techByKey`, cycle-guarded). For each
  ancestor record its **depth** (min hops from the selected tech) and whether it
  is **hidden** (`!active.has(category)`). Only mount the panel if `some ancestor
  is hidden`.
- `AncestryPanel` renders a **fixed overlay** positioned to the LEFT of the
  selected card's on-screen rect (pass the card's DOMRect like the tooltip does;
  clamp to viewport; scrollable if tall). Layout as **depth columns**: the
  selected tech on the far right, depth-1 prereqs in the next column left,
  depth-2 further left, etc. Each column stacks that depth's ancestors as
  **mini-cards** (icon + name, area-colored left border). Draw simple SVG/border
  connector lines between a prereq mini-card and the card(s) it feeds (right
  neighbor). **Hidden** ancestors get a visible marker (e.g. a dashed border + a
  small "hidden" dot) so you can tell which the filter removed.
- Interactivity: each mini-card is hoverable → reuse the existing hover pathway
  to show `TechTooltip` (pass the same `onEnter/onLeave` used by TechCard, so
  hovering a panel mini-card shows its full tooltip); clicking a mini-card calls
  `onSelect(key)` → re-roots the drill-down on that ancestor.
- Wire into TechTree next to the tooltip render. Reuse `techByKey`, `iconBase`,
  `CATEGORY_AREA`. Plain-text names (D-05).

**Verify:** dev server — filter out a category, select a tech that depends on it:
a left panel shows the full ancestry with the hidden prereqs marked; hovering a
mini-card shows its tooltip; clicking one re-roots. With nothing hidden, no panel.
