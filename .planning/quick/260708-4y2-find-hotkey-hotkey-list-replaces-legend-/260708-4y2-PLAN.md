# Quick Task 260708-4y2 — Find + hotkey list + weight-modifier detail

Serves the #1 use case: search a tech → jump to it → see its unlock path (already
built: select/highlight/ancestry) + what boosts your chance of finding it.

## Task 1 — F-Find (search → select + jump)

- NEW `app/src/components/FindOverlay.tsx`: a centered search overlay. Autofocused
  text input; filters ALL techs by name (case-insensitive `includes`), shows the
  top ~12 matches as rows (icon + name + `Category · Tier`, area-colored left
  border). Keyboard: ↑/↓ move the active row, Enter picks it, Esc closes; rows are
  also clickable. Props: `techs: {key,name,category,tier,area,icon}[]`,
  `iconBase`, `onPick(key)`, `onClose()`.
- `TechTree`: add `findOpen` state. A window `keydown` effect: when the key is
  `f`/`F` with NO modifiers AND the event target is not an input/textarea/
  contenteditable AND `!findOpen` → `preventDefault()` + open. Render
  `<FindOverlay>` when open.
- `onPick(key)`: `setSelectedKey(key)`; if the tech's `category[0]` is NOT in
  `active`, add it (`setActive`) so its card renders; `fitToNodes` to that node
  (look it up in `state.layout.nodes`) to pan/zoom-jump to it; close the overlay.
  Search must include HIDDEN techs so you can find + reveal a filtered-out tech.
- Build a searchable list `useMemo` from `snapshot.techs`.

**Verify:** dev server — press F, type part of a tech name, Enter/click a result:
the tree jumps to it, it's selected (gold), and if it was filtered out its
category is re-enabled. Esc closes. Typing 'f' inside the find box does not
re-trigger.

## Task 2 — Remove the Legend, add a hotkey list to the left panel

- Remove `<Legend />` from `TechTree` (and delete `Legend.tsx` + its CSS, and the
  Legend import). The left nav already lists every category with area-colored
  headers, so the legend box is redundant.
- `CategoryNav`: make it a flex column — the category groups scroll (`flex: 1 1
  auto; overflow-y: auto`) and a NEW `.category-nav__hotkeys` block is pinned at
  the BOTTOM (`flex: 0 0 auto`, top border). It lists the shortcuts as compact
  `key — action` rows: **F** — Find · **Esc** — Deselect / close · **Click** —
  Select · **Drag** — Pan · **Wheel** — Zoom. Style the key in a small
  keycap-ish badge. Plain text.

**Verify:** dev server — the bottom-left legend box is gone; a "Shortcuts" list
sits pinned at the bottom of the left panel; the category list still scrolls
above it.

## Task 3 — Weight-modifier detail ("what boosts find-chance")

- NEW pure helper `app/src/lib/graph/weight.ts`:
  `describeWeightModifiers(raw: unknown, techByKey: Map<string,Tech>): string[]`.
  `weightModifierRaw` shape: an object that may have a top-level `factor` (base
  multiplier) and a `modifier` field (an array OR a single object); each modifier
  entry has a `factor` plus one or more CONDITION keys. Produce readable lines:
  - top-level `factor` (if present and ≠ 1) → `"Base ×{factor}"`.
  - each modifier → `"×{factor} if {conditions}"`.
  - condition formatting: `has_technology: "tech_x"` → `researched {resolved
    name}` (via techByKey; fall back to the key). Generic condition
    `key: value` → humanize the key (snake_case → words) + value; if value is an
    object like `{GREATER_THAN: 2}` render `> 2` (map GREATER_THAN→>, LESS_THAN→<,
    etc.); join multiple conditions with " and ". Be defensive (unknown shapes →
    skip, never throw).
- `TechTooltip`: add a **"Research Weight"** section (after the meta / before or
  after Unlocks) showing base `Weight {tech.weight}` and, if
  `describeWeightModifiers` returns any lines, list them (bulleted). Pass
  `techByKey` (already available). Section header styled like the others.
- Unit test `weight.test.ts` for `describeWeightModifiers` (the thrusters
  has_technology example → "×2 if researched …"; the num_owned_planets example →
  "> 2"; empty/undefined → []).

**Verify:** dev server — hovering a tech with weight modifiers (e.g. one with
`has_technology` boosts) shows a "Research Weight" section listing the base weight
+ readable "×N if researched X" lines.
