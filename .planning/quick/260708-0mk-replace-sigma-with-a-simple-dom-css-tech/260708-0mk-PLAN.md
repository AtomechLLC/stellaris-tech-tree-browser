# Quick Task 260708-0mk — Reference-styled DOM tech tree

## Reference (what we're matching)

`draconas1/stellaris-tech-tree` `www/` uses **vis.js**: hierarchical layout
`direction:"LR"`, `sortMethod:"directed"`, `levelSeparation:600`,
`nodeSpacing:120`, physics OFF; nodes are area-colored `box` cards (icon on the
left, name + `Category · Tier` + `Cost/Weight`); edges are dashed cubic-bezier
prerequisite connectors, no arrows; dark theme; area filter tabs
(All/Physics/Society/Engineering) + search.

## Our approach (DOM, not vis.js — user's choice)

Reuse **ELK** (elkjs, already a dep) for the hierarchical LR layout AND edge
routing, then render pure DOM:

- ELK `layered`, `direction: RIGHT` (tier 0 left → tier 5 right), tier pinned via
  partitioning (as today), **`edgeRouting: ORTHOGONAL`** so ELK returns elbow bend
  points for connectors, node width/height = the real card size (so ELK spaces
  cards with no overlap). NO area-band Y-remap (let ELK optimize crossings, like
  the reference) — area is conveyed by card COLOR, not forced bands.
- Render inside `.tree-canvas` (one element with `transform: translate(pan)
  scale(zoom)`): absolutely-positioned DOM `.tech-card`s at each node's ELK (x,y),
  and an `<svg>` edge layer drawn from ELK's edge bend points. Cards + SVG share
  the canvas coordinate space, so pan/zoom is ONE css transform — no per-frame
  sync, no projection, no culling.
- **Pan** = drag (mousedown/move → translate); **zoom** = wheel + buttons (scale,
  clamped). Standard ~30-line transform handler.
- **Cards** = reference-style, area-colored, rich HTML (full control): icon +
  tier badge, area-colored header bar with the name, `Category · Tier N`,
  `Cost: X, Weight: Y`, subtle background texture, area-colored border.
- **Area filter tabs** (All/Physics/Society/Engineering) filter which nodes+edges
  render (re-run or just hide+re-layout on change). **Dark theme** canvas.

No Sigma, no WebGL, no camera. This is verifiable by the orchestrator via normal
screenshots (pure DOM).

## Data facts

Every tech has one `category[0]`; 13 categories in 3 areas; tiers 0–5; card fields
`name, category, tier, cost, weight, area, icon`; 613 prerequisite pairs, 88

## Task 1 — ELK layout from tech.json → positions + edge routes

**Files:** NEW `app/src/lib/tree/layoutTree.ts`; retire the Sigma-coupled
`app/src/lib/graph/{buildGraph,layout,swimlanes,areaBands}.ts`; keep `categories.ts`,
`theme.ts`.

**Action:** `layoutTree(snapshot, cardW, cardH)` builds an ELK graph directly from
`snapshot.techs`: one ELK child per tech (`width:cardW, height:cardH`,
`layoutOptions: { "elk.partitioning.partition": String(tier) }`); one ELK edge per
prerequisite pair where both techs exist (guard dangling like buildGraph did).
Root layoutOptions: `elk.algorithm:"layered"`, `elk.direction:"RIGHT"`,
`elk.partitioning.activate:"true"`, `elk.separateConnectedComponents:"false"`,
`elk.layered.thoroughness:"1"` (keep the perf tuning from the old layout.ts),
`elk.edgeRouting:"ORTHOGONAL"`, generous `elk.layered.spacing.nodeNodeBetweenLayers`
(~120) and `elk.spacing.nodeNode` (~24). Return `{ nodes: Array<{key, x, y, w, h,
tech}>, edges: Array<{from, to, sections}>, width, height }` where `sections` is
ELK's edge bend-point routing (`edge.sections[].startPoint/bendPoints/endPoint`).
Reuse the in-process `elk.bundled.js` instance (Pitfall 4 — no real worker).

**Verify:** a small unit test: every tech gets a positioned node; x increases with
tier (tier 0 leftmost); edges only reference existing techs; `npm run build`.
**Done:** ELK positions + orthogonal edge routes available to the renderer.

## Task 4 — SVG connector edges (from ELK routes)

**Files:** NEW `app/src/components/EdgeLayer.tsx`; CSS.

**Action:** one `<svg class="edge-layer">` inside `.tree-canvas`, width/height =
layout size, `pointer-events:none`, z-index BELOW cards. For each edge, build a
`<path>` (or polyline) from its ELK `sections` bend points
(startPoint → bendPoints… → endPoint) → clean orthogonal elbow connectors like the
reference. If an edge has no sections, fall back to a straight/elbow line from the
source card's right-center to the target's left-center. Stroke = light token
(`--color-edge`), thin, optionally dashed (reference uses dashes). Because the SVG
is inside the transformed canvas, it pans/zooms with the cards automatically — no
recompute on pan/zoom.

**Verify:** dev server — dashed elbow lines connect prerequisites, aligned to
cards, and stay aligned through pan/zoom.
**Done:** always-on reference-style connectors.
